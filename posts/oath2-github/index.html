Mendapatkan Token Oath2 Github menggunakan Web Browser dan Postman

Diterbitkan 04 November 2020 | 2 menit membaca | 318 kata | Adittia Kurniawan

Login ke akun github anda lalu buka Developer settings untuk membuat aplikasi oath2 baru. Kemudian buat aplikasi baru dan isi data yang diperlukan

Kita akan menggunakan http://127.0.0.1/callback sebagai Authorization callback URL agar saat authentikasi berhasil kita akan dialihkan ke alamat yang aman (localhost).

JANGAN menggunakan alamat yang tidak aman sebagai Authorization callback URL. Gunakanlah alamat yang aman seperti http://127.0.0.1/callback

Setelah berhasil mendaftarkan aplikasi oath2 kita akan dialihkan ke halaman pengaturan aplikasi. Di halaman ini buathlah client_secret baru dengan cara klik generate a new client secret. Simpanlan client secret tersebut di tempat yang aman karena kita tidak bisa mendapatkan client secret kita kembali nanti.

Setelah mendapatkan client_id dan client_secret, buka web browser favorite anda kemudian kunjungi alamat berikut ini https://github.com/login/oauth/authorize?client_id=80695ce3635f875a3747&redirect_uri=http://127.0.0.1/callback&state=mustberandom Silahkan login dan authorize menggunakan akun github anda.

Setelah berhasil anda akan di alihkan ke halaman web yang telah anda daftarkan sebagai Authorization callback URL.

Setelah berhasil login kita akan mendapat dua buah parameter yang penting dari url diatas yaitu code dan state. Untuk sekarang kita dapat mengabaikan parameter state, namun dalam aplikasi sesungguhnya kita harus mengecek apakah nilai parameter state sama dengan nilai yang kita kirimkan sebelum login.

Selanjutnya kita akan melakukan request sekali lagi untuk mendapatkan access_token yang sebenarnya. Buka postman dan lakukan POST request ke halaman https://github.com/login/oauth/access_token. Masukan informasi yang diperlukan pada body dengan fromat x-www-form-urlencoded.

Kita telah berhasil memperoleh access_token. Kita dapat mencoba menggunakan access_token tersebut dengan melakukan GET request pada alamat https://api.github.com/user.

Cara yang digunakan dalam artikel ini hanya menunjukan tentang proses mendapatkan token oath2 pada github. JANGAN pernah membagikan client_secret anda ke orang lain dan JANGAN gunakan cara ini pada penggunaan aplikasi sesunggunya. Jika anda ingin mengimplementasikan oath2 pada aplikasi sesungguhnya lebih baik anda menggunakan library yang stabil dan terpercaya.
